Learning with Imperfect Data

Posted on Wed, Aug 17, 2022 Lecture Notes Federated Learning Long-Tail Learning Noisy Label Learning Continual Learning

The following notes are summarized from https://www.bilibili.com/video/BV1oW4y1U7Tr?share_source=copy_web&vd_source=48a82fd721223ae2fbed15ddf55eb2c2, the contents are not fully complete.

Federated Learning: Data Is Not Centralized

  • To aggregate the models trained on the distributed data from different clients, instead of the distributed data themselves.

Long-Tail Learning: Data Is Class Imbalanced

  • Re-weighting: to assign higher weights for minority classes in the loss function so that the model is less likely to make wrong predictions on the minority.
  • Augmentation / over-sampling: to increase the amount of the minority so that the class can be balanced.
  • Ensembling: to ensemble multiple models trained on different data distribution.
  • Decoupling: to train on the original distribution in the feature extraction stage and adopt the above tricks in the classification stage.

Noisy Label Learning: Data Is Mislabeled

  • Co-teaching: to train two models, where each model is trained with clean samples selected by the other model.
  • Re-weighting: to reduce the weight of noisy samples in the training loss.

Continual Learning: Data Is Gradually Coming

A trade-off between stability (the maintenance on the old task) & plasticity (the adaptation to the new task).

  • Replay: to save the most representative data in the past for future training.
  • Regularization: to save the historical model parameters for future training.
